# Storybook Lit Component Library

This is a generic web component library built with [Lit](https://lit.dev/) and documented using [Storybook](https://storybook.js.org/).
It is designed to be published as a Node module and reused across different projects in your organization.

## Features

- **Lit Framework:** Lightweight, expressive, fast base class for Web Components.
- **Storybook Setup:** Browse and interact with your components in isolation.
- **Vite Build System:** Fast and optimized build for distribution.
- **TypeScript:** Fully typed components, providing excellent IDE support.

## Project Structure

- \`src/\`: Contains component source code (\`*.ts\`) and Storybook stories (\`*.stories.ts\`).
- \`dist/\`: The compiled output generated by Vite, containing CommonJS, ES Modules, and TS declaration files.
- \`.storybook/\`: Configuration for Storybook.

## Commands

- \`npm run build\`: Compiles TypeScript and builds the library with Vite into the \`dist\` folder.
- \`npm run storybook\`: Starts Storybook locally on port 6006 to view and develop your components.
- \`npm run build-storybook\`: Builds a static Storybook app for deployment to CI/CD environments.

## How to use the Component Library

Once the package is published to your organization's registry or linked via npm, you can install it in a consuming project:

\`\`\`bash
npm install storybook-lit
\`\`\`

Then, import the component in your JS/TS application:

\`\`\`javascript
import 'storybook-lit/my-element'; // This will register the custom element
\`\`\`

And use it in your HTML:

\`\`\`html
<my-element name="Organization" count="5"></my-element>
\`\`\`

## CI/CD Publishing

Make sure your CI/CD pipeline runs \`npm run build\` before running \`npm publish\`.
The \`package.json\` is already configured to export the compiled bundles and type declarations from the \`dist/\` folder.
